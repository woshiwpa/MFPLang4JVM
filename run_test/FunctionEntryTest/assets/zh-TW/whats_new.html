程式開發科學計算器的新特性

程式開發科學計算器已經在百度貼吧安家了，請到百度MFP吧參加對程式開發科學計算器以及MFP程式開發語言的討論以獲得最快捷的幫助和最新的資訊。

1.6.5版

1. 二維圖形表達式繪圖引擎在1.6.5.48中得到顯著改進。由於引入緩存技術以及對拖動事件響應進行了優化，現在計算器對圖形各種操作更加快速和敏感。但要註意，由於對表達式繪圖必須在新的值計算表達式結果，如果表達式過於復雜，用戶仍然會感覺到明顯的延遲。

2. 支持繪制2維或3維隱函數圖形。註意只有當隱函數可解，才能夠繪制圖形。4次或者4次以上的多項式由於根表達式過於復雜，雖然可解，但無法繪圖。。如果繪制包含兩個變數的隱函數，最多4個根表達式將會被繪制出。如果是3個變數的隱函數，根據情況計算器可能會對每個變數求根，然後為每個變數繪制出最多2個解表達式。這樣一來，一個單一的3變數的隱函數，可能意味著繪制出6條圖形，整個求解繪圖過程將會花費比較長的時間。還要註意，隱函數必須是等式，而不能是賦值式，比如x**2+y**2=4-z**2是非法的，而x**2+y**2==4-z**2則是正確的；

3. 函數程式size，zeros和ones的返回值和參數定義有所改變。現在，如果是計算一個數值，而不是陣列的尺寸，size函數程式返回[]而不是[0]。同理，如果參數是[]，zeros返回0而ones返回1；

4. 一定程度的性能提升；

5. 很多錯誤得到了修正；

1.6.4版

1. 1.6.4.46版在數學表達式識別正確率和速度方面有了進一步的提升，此外不定積分功能也進一步加強；

2. 加入數學公式拍照識別的功能。用戶可以進入智慧計算器後點擊照相機圖標按鈕進入數學公式拍照識別。註意現階段只支持打印體數學公式識別，手寫體識別還在發展完善中；

3. 提供了Unicode字符串和整數數組相互轉換的函數程式conv_str_to_ints以及conv_ints_to_str；

4. 提供了system函數程式直接運行一個系統命令；

5. 允許用戶隱藏3維圖形的坐標軸和標題；

6. 一些小的錯誤的修正；

1.6.3版

1. 從1.6.3.44開始對啟動智慧計算器和命令提示符時預定義程式庫裝載過程進行了優化，啟動速度大大加快；

2. 提供了一整套完整的檔案操作函數程式包括create_file、copy_file、delete_file、cd、dir、pwd、fopen、fclose、fprintf、fscanf、freadline、fread、fwrite等等。現在命令提示符完全可以當成安卓設備的一個外殼程式（shell）來使用。用戶可以在裏面對安卓設備中的任何檔案進行讀寫操作，只要用戶有這個權限。當然，這項新功能雖然強大，但也很危險。強烈建議用戶把所有的檔案操作限制在應用的檔案目錄（AnMath）中。同時，建議用戶在應用的檔案目錄（AnMath）下創建一個data檔案目錄用於保存個人數據，以方便管理；

3. 提供了計算陣列的秩的rank函數程式；

4. 提供了新的輸入輸出函數程式包括printf、scanf、sprintf、sscanf；

5. 提供了新的字串操作函數程式包括split、trim、trim_left和trim_right；

6. 提供了pause函數程式用於暫停運行中的程式，用戶可以敲擊Enter鍵繼續；

7. 提供了一整套日期和時間的函數程式；

8. 一些小的錯誤的修正；

1.6.0版

1. 更快的計算速度。經過測試，基於JAVA的程式開發科學計算器在電腦上計算速度是以前各版本的10倍，在安卓設備上，由於速度瓶頸未必是在計算上，並且用戶通常一次只運行一個小的命令，對計算速度的提高，用戶感受可能並非非常明顯。但對於大部分計算，速度仍然可以達到以前的1.5到3倍，具體取決於安卓設備的硬體條件；

2. MFP程式開發語言現在可以處理任意大的數，比如，如果計算1000的1000次方，以前的版本會報溢出錯誤，現在用戶可以得到結果為10的3000次方；

3. 計算結果更精確。著名的類Matlab數學軟體Octave采用雙精度數進行內部計算所以其實際精度只能達到約10E-17。但MFP則可以達到5*10E-48。當數值小於這個值時，MFP會視該值為0；

4. 支持無窮大和NaN。由於這項新的特征，本軟體將不會再拋出除0異常。用戶必須自己在代碼中處理。MFP提供了一系列的新程式，比如is_nan_or_inf、is_inf、includes_nan以及includes_inf。使用這些程式用戶可以輕松判斷一個數是否是無窮或者NaN，或者一個復數或者陣列是否包括無窮或者NaN。用戶也可以用系統預定義的變數inf和nan來生成無窮和NaN值並進行計算；

5. 計算特征值和特征向量。註意這是非常消耗內存和CPU資源的計算。強烈建議，在安卓設備上不要計算超過6*6的矩陣，在個人電腦不要計算超過8*8的矩陣。否則可能會內存不足或者用戶陷入長時間等待；

6. 支持自定義智慧計算器和命令提示符的鍵盤。用戶現在不但可以自定義函數程式鍵盤，還可以自定義數字和運算符鍵盤。用戶還可以自由改變按鍵高度，寬度以及按鍵上的字的顏色；

7. 一些其他小的改進和錯誤修正；

1.5.6版

1. 完全支援2進制，8進制和16進制整數和小數。2進制由0b起頭，比如0b0011101和0b11.1001。8進制由一個額外的0起頭，比如02751和00.23561。16進制0x起頭，比如0xEFF6和0x23FAB.DF24。此外，還提供了新的進制轉換程式。原有的進制轉換程式，也就是int*to*（比如int10to2以及int2to16），被新的conv_*_to_*（比如conv_hex_to_dec）程式所取代，並將在1.6版中被移除。新的進制轉換程式全部支持整數和小數；

2. 增加新的程式sum_over和product_over；

3. 改進了智慧計算器的自定義輸入面板；

4. 改善了主界面；

5. 為避免和if語句混淆，將if程式重命名為iff；

6. 一些其他小的改進和錯誤修正；

1.3.1版

1.本軟體增加了顯示啟動狀態的功能。程式開發科學計算器啟動時需要將內置的，預定義的和用戶定義的函數庫載入，需要將預定義的函數拷貝到存儲卡的例子文件夾中，還需要將基於JAVA的，可運行於任何安裝了JAVA的Windows、MacOSX和Linux的程式開發科學計算器拷貝到存儲卡中（此操作僅僅當用戶安裝或升級本軟體時才會進行），如果移動設備硬件性能較低，啟動過程可能需要花費比較長的時間。如果啟動過程太長，界面不再響應，用戶可能會認為軟體已經死機。而這一新的特性將可以告訴用戶啟動時本軟體在做什麽；

2.對界面作了一些小的改進。

1.1版

1.本軟體增加了自帶了基於JAVA的，可運行於任何安裝了JAVA支持的Windows、MacOSX和Linux的程式開發科學計算器。用戶只需要用一根USB連接線連接移動設備和個人電腦，在移動磁盤中找到 AnMath子目錄，進入子目錄並在個人電腦中運行JMathCmd.jar。用戶可以在個人電腦中編寫和測試程序並存入移動設備中，以便以後在移動設備中使用；

2.加強了對三角，雙曲三角和指數對數函數的復數支持，換句話說， 類似於sin(3 + 4i)、atan(0.9-47.33i)和log(-7-9.2*i)都是合法的了；
