What's new in Scientific Calculator Plus

A free app, Smart Photographic Calculator, has been released by CYZ Soft. This app is a simplified version with all the capabilities of Scientific Calculator Plus except programming. It is much easier to use for students. Please download from google play and try it.

v 1.6.5

1. 2D graph engine for expressions is improved in 1.6.5.48. Caching technology is implemented to store calculated results. And the expression chart is more sensative to Drag-move operations. However, note that plotting expressions is different from plotting data. Expression result at a new value need be calculated anyway. As such if the expression is not simply, user may still experience significant lag.

2. Plot graph for 2-variable and 3-variable implicit functions. Note that only solvable implicit functions can be plotted. Polymonials of degree larger than 3 can be solved but cannot be plotted because the root expressions are very complicated. If plot 2-variable implicit function, at most 4 root expressions can be drawn. For 3-variable implicit function, Calculator may calculate roots for each of the variables and then plot at most 2 roots for every variable. In other words, a single 3-variable implicit function may imply at most 6 to-be-plotted expressions. The whole solving-plotting procedure in this case will take quite long time. Also note that implicit function must be an equation, not assignment, i.e. x**2+y**2=4-z**2 is invalid while x**2+y**2==4-z**2 can be accepted;

3. Change behavior of functions size, zeros and ones. Now if calculate size of a value, not matrix, [] not [0] will return. Also, function zeros and ones return 0 and 1 respectively if input parameter is [];

4. Performance improvement;

5. A number of bug fixes;

v 1.6.0

1. Faster calculation. Now Scientific Calculator Plus for JAVA is running ten times faster in PC. In Android devices, since the bottleneck is not necessarily script analysis and calculation, and usually only a single small command is run each time, user may not necessarily have a very strong feeling of accelerated calculation. However, in most cases, Scientific Calculator Plus is about 1.5-3 times faster when doing mathematic analysis in a mobile device, depending on the device's handware.

2. Now MFP is able to process any large values. For example, before v.1.6, MFP got double overflow when calculate 1000**1000. Now, user can see the result is 10**3000. This is even Octave cannot achieve;

3. Result is more accurate. Octave simply uses double in its calculation while MFP does not. As such, the rounding error of MFP is only 5*10E-48 compared to Octave which can be as large as 10E-17. Only if a value is smaller than this MFP cannot handle (MFP will asssume it is zero);

4. Support infinite and nan. Because this new feature, Scientific Calculator does not throw any divided by zero exception from this release. Users have to handle this issue by themselves in the source code. MFP has provided a number of new functions like is_nan_or_inf, is_inf, includes_nan and includes_inf to identify if a value is nan or inf or an array or complex number includes nan or inf. User can also use nan and inf as system pre-defined variables;

5. Calculate eigen values and vectors. Note that eigen values calculation is very time consuming and needs big memory. It is strongly recommended that user does not run more than 6*6 matrix eigen values/vectors calculation in a mobile device and does not run more than 8*8 matrix eigen values/vectors calculation in a PC. Otherwise, user may run out of memory or wait for ever;

6. Support user defined input pad for both smart calculator and command line. Moreover, user is able to define numbers and operators, set key width and height so that make button smaller or bigger. And user is also be able to set key text color;

7. Minor functionality optimized and several bugs fixed;
