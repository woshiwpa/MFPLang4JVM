可编程科学计算器的新特性

可编程科学计算器已经在百度贴吧安家了，请到百度MFP吧参加对可编程科学计算器以及MFP编程语言的讨论以获得最快捷的帮助和最新的信息。

1.6.5版

1. 二维图形表达式绘图引擎在1.6.5.48中得到显著改进。由于引入缓存技术以及对拖动事件响应进行了优化，现在计算器对图形各种操作更加快速和敏感。但要注意，由于对表达式绘图必须在新的值计算表达式结果，如果表达式过于复杂，用户仍然会感觉到明显的延迟。

2. 支持绘制2维或3维隐函数图形。注意只有当隐函数可解，才能够绘制图形。4次或者4次以上的多项式由于根表达式过于复杂，虽然可解，但无法绘图。。如果绘制包含两个变量的隐函数，最多4个根表达式将会被绘制出。如果是3个变量的隐函数，根据情况计算器可能会对每个变量求根，然后为每个变量绘制出最多2个解表达式。这样一来，一个单一的3变量的隐函数，可能意味着绘制出6条图形，整个求解绘图过程将会花费比较长的时间。还要注意，隐函数必须是等式，而不能是赋值式，比如x**2+y**2=4-z**2是非法的，而x**2+y**2==4-z**2则是正确的；

3. 函数size，zeros和ones的返回值和参数定义有所改变。现在，如果是计算一个数值，而不是矩阵的尺寸，size函数返回[]而不是[0]。同理，如果参数是[]，zeros返回0而ones返回1；

4. 一定程度的性能提升；

5. 很多错误得到了修正；

1.6.0版

1. 更快的计算速度。经过测试，基于JAVA的可编程科学计算器在电脑上计算速度是以前各版本的10倍，在安卓设备上，由于速度瓶颈未必是在计算上，并且用户通常一次只运行一个小的命令，对计算速度的提高，用户感受可能并非非常明显。但对于大部分计算，速度仍然可以达到以前的1.5到3倍，具体取决于安卓设备的硬件条件；

2. MFP编程语言现在可以处理任意大的数，比如，如果计算1000的1000次方，以前的版本会报溢出错误，现在用户可以得到结果为10的3000次方；

3. 计算结果更精确。著名的类Matlab数学软件Octave采用双精度数进行内部计算所以其实际精度只能达到约10E-17。但MFP则可以达到5*10E-48。当数值小于这个值时，MFP会视该值为0；

4. 支持无穷大和NaN。由于这项新的特征，本软件将不会再抛出除0异常。用户必须自己在代码中处理。MFP提供了一系列的新函数，比如is_nan_or_inf、is_inf、includes_nan以及includes_inf。使用这些函数用户可以轻松判断一个数是否是无穷或者NaN，或者一个复数或者矩阵是否包括无穷或者NaN。用户也可以用系统预定义的变量inf和nan来生成无穷和NaN值并进行计算；

5. 计算特征值和特征向量。注意这是非常消耗内存和CPU资源的计算。强烈建议，在安卓设备上不要计算超过6*6的矩阵，在个人电脑不要计算超过8*8的矩阵。否则可能会内存不足或者用户陷入长时间等待；

6. 支持自定义智慧计算器和命令提示符的键盘。用户现在不但可以自定义函数键盘，还可以自定义数字和运算符键盘。用户还可以自由改变按键高度，宽度以及按键上的字的颜色；

7. 一些其他小的改进和错误修正；

2. 增加新的函数sum_over和product_over；

3. 改进了智慧计算器的自定义输入面板；

4. 改善了主界面；

5. 为避免和if语句混淆，将if函数重命名为iff；

6. 一些其他小的改进和错误修正；
